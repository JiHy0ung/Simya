# 생선살 기대단가(평균가) 계산 시스템 구현 요청

## 목표

낚시 페이지(`src/pages/Fish/FishPage.jsx`)에서 사용자가 보유한 낚싯대 등급을 입력받아 localStorage에 저장하고, 이를 기반으로 **사이즈별 생선살의 "기대 단가"** 를 계산하여 localStorage에 캐싱한다. 이 기대단가는 이후 요리/가공 페이지에서 재료(생선살)의 원가로 사용된다.

## 0. 작업 전 확인사항

- `src/constants/town/town.js` — 마을/물고기 정보 참고
- `src/data/fish.js` — 물고기 가격/등급/사이즈 데이터
- 이전에 작성한 `FishPage.jsx` 관련 작업(낚시 도감 페이지) 및 분해 로직(`getFishDecomposition` 등)이 이미 있다면 해당 유틸을 재사용/확장한다.
- 요리/가공 페이지가 이미 존재한다면 해당 페이지를 찾아 구조를 참고하고, 생선살 가격을 어디서 가져다 쓰는지 확인한다.

## 1. 배경 개념

### 1-1. 분해 시 생선살 개수 (등급별, 기존 규칙 재확인)

| 등급 | 분해 개수 |
| ---- | --------- |
| 일반 | 1개       |
| 희귀 | 2개       |
| 고급 | 4개       |
| 특급 | 6개       |
| 전설 | 10개      |

### 1-2. 마을별 사이즈별 물고기 가격 (등급별)

사이즈가 같으면 마을이 달라도 가격이 동일하다 (위키 기준 검증됨).

| 사이즈 | 일반 | 희귀 | 고급 | 특급 | 전설 |
| ------ | ---- | ---- | ---- | ---- | ---- |
| 소형   | 300  | 500  | 850  | 2000 | 4000 |
| 중형   | 500  | 750  | 1300 | 2500 | 7000 |
| 대형   | 700  | 1200 | 1800 | 3500 | 9000 |

### 1-3. 낚싯대 등급별 물고기 출현 확률 (등급 확률, 사이즈 무관 공통)

| 낚싯대                 | 일반 | 희귀 | 고급 | 특급 | 전설 |
| ---------------------- | ---- | ---- | ---- | ---- | ---- |
| 행상의 낡은 낚싯대     | 0.75 | 0.20 | 0.05 | 0    | 0    |
| 빛바린 상회의 낚싯대   | 0.58 | 0.26 | 0.13 | 0.03 | 0    |
| 부서진 만월의 낚싯대   | 0.43 | 0.31 | 0.19 | 0.06 | 0.01 |
| 희미한 별빛의 낚싯대   | 0.26 | 0.30 | 0.25 | 0.15 | 0.04 |
| 잊혀진 월광의 낚싯대   | 0.10 | 0.22 | 0.31 | 0.24 | 0.13 |
| 깊어지는 심야의 낚싯대 | 0    | 0.10 | 0.30 | 0.34 | 0.26 |
| 고요한 영겁의 낚싯대   | 0    | 0    | 0.25 | 0.35 | 0.40 |

크기(소형/중형/대형) 출현 확률은 60% / 30% / 10%로 모든 낚싯대 공통이지만, **이번 기대단가 계산에는 사이즈 확률은 쓰지 않는다** (이미 사이즈별로 따로 계산하므로). 등급 확률만 가중치로 사용한다.

## 2. 기대단가 계산 공식

특정 **사이즈**에 대해, 사용자가 보유한 낚싯대의 등급 확률을 가중치로 사용해 다음과 같이 계산한다:

```
기대단가(사이즈) = Σ [ P(등급) × 가격(사이즈, 등급) ] / Σ [ P(등급) × 분해개수(등급) ]
```

- 분자: 등급별 (출현확률 × 물고기가격)의 합
- 분모: 등급별 (출현확률 × 분해개수)의 합
- 즉 "이 낚싯대로 해당 사이즈 물고기를 낚아 분해했을 때, 생선살 1개당 평균적으로 가지는 가치"

### 계산 예시 (검증용)

"행상의 낡은 낚싯대"(일반0.75, 희귀0.20, 고급0.05, 특급0, 전설0)로 소형 생선살 기대단가 계산:

- 분자 = 0.75×300 + 0.20×500 + 0.05×850 = 225 + 100 + 42.5 = 367.5
- 분모 = 0.75×1 + 0.20×2 + 0.05×4 = 0.75 + 0.4 + 0.2 = 1.35
- 기대단가 = 367.5 / 1.35 ≈ **272.2G**

이 값으로 유틸 함수 작성 후 단위 테스트나 콘솔 로그로 검증한다.

### 결과 형태

마을 구분 없이 **사이즈 3종 × 낚싯대 7종 = 21가지** 조합이 가능하지만, 실제로는 **사용자가 선택한 낚싯대 1개에 대해서만 사이즈 3종(소형/중형/대형)의 기대단가 3개**를 계산하면 된다.

```js
{
  소형: 272.2,
  중형: ...,
  대형: ...,
}
```

## 3. UI / UX

### 3-1. 낚싯대 등급 입력

- `FishPage.jsx`에 낚싯대 등급을 선택하는 UI 추가 (select, 드롭다운, 또는 버튼 그룹 — 기존 디자인 시스템의 select/버튼 컴포넌트 재사용).
- 선택지 7개: 행상의 낡은 낚싯대 / 빛바린 상회의 낚싯대 / 부서진 만월의 낚싯대 / 희미한 별빛의 낚싯대 / 잊혀진 월광의 낚싯대 / 깊어지는 심야의 낚싯대 / 고요한 영겁의 낚싯대
- 페이지 진입 시 localStorage에 저장된 값이 있으면 자동으로 불러와 선택 상태를 복원한다.
- 사용자가 낚싯대를 변경하면:
  1. localStorage에 선택한 낚싯대 정보 저장
  2. 사이즈별 기대단가 즉시 재계산
  3. 계산 결과도 localStorage에 캐싱 저장 (낚싯대를 바꾸지 않는 한 재계산하지 않고 캐시된 값 사용)

### 3-2. localStorage 저장 구조 (제안)

```js
// key: "fishRodSettings"
{
  rodGrade: "잊혀진 월광의 낚싯대", // 사용자가 선택한 낚싯대명
  expectedFishMeatPrice: {
    소형: 1234.5,
    중형: 2345.6,
    대형: 3456.7
  },
  updatedAt: "2026-06-20T12:00:00.000Z" // 캐싱 갱신 시각 (선택)
}
```

- 키 이름은 프로젝트의 기존 localStorage 네이밍 컨벤션이 있다면 그것을 따른다 (예: camelCase, prefix 등 기존 패턴 확인).
- 낚싯대 변경 시에만 재계산 + 캐시 갱신. 동일 낚싯대로 페이지 재방문 시에는 캐시된 `expectedFishMeatPrice`를 그대로 읽어서 사용 (재계산 불필요).

### 3-3. 화면 표시

- 선택한 낚싯대와 함께, 계산된 사이즈별 생선살 기대단가를 표 또는 카드 형태로 보여준다 (기존 페이지의 정보 카드/표 스타일 재사용).
- 예: "소형 생선살 평균가: 272G", "중형 생선살 평균가: ...", "대형 생선살 평균가: ..."

## 4. 요리/가공 페이지 연동

- 요리/가공 레시피에 들어가는 생선살 재료의 원가는, localStorage에 캐싱된 `expectedFishMeatPrice[사이즈]` 값을 가져와 `재료 개수 × 기대단가`로 계산한다.
- 마을 구분이 없으므로 레시피 재료가 "OO 소형 생선살"처럼 마을명이 붙어있어도, 가격 계산 시에는 사이즈만 추출해서 사용한다 (예: "세레니티 소형 생선살" → 사이즈="소형"으로 매칭).
- **낚싯대 미설정 시 fallback 평균값**: 사용자가 아직 낚싯대를 선택하지 않아 localStorage에 캐시된 기대단가가 없는 경우, 아래의 고정 fallback 값을 사용한다. 이 값은 **7개 낚싯대 전체의 사이즈별 기대단가를 단순평균**하여 미리 계산해 둔 상수다.

  | 사이즈 | fallback 평균단가 |
  | ------ | ----------------- |
  | 소형   | 298.2G            |
  | 중형   | 460.1G            |
  | 대형   | 638.9G            |

  (참고: 위 수치는 7개 낚싯대 각각의 사이즈별 기대단가— `Σ[P(등급)×가격(사이즈,등급)] / Σ[P(등급)×분해개수(등급)]` —를 구한 뒤, 7개 값을 단순 산술평균한 것. 상수 데이터 작성 시 동일한 계산 함수로 재검증 후 하드코딩하거나, 페이지 최초 로드시 동적으로 7개 낚싯대 평균을 계산해 fallback으로 사용해도 무방하다.)

- UI 표시 시, fallback 값을 쓰는 중이라면 "평균 시세 기준 (낚싯대를 설정하면 내 낚싯대 기준으로 정확하게 계산됩니다)" 같은 안내 문구를 함께 보여줘 사용자가 어떤 기준의 가격인지 알 수 있게 한다.
- 낚싯대를 설정한 사용자는 fallback 대신 본인의 캐싱된 기대단가를 우선 사용한다.

## 5. 구현 순서 제안

1. 등급 확률 / 사이즈별 가격 / 분해 개수 상수를 정리한 데이터 파일 작성 (예: `src/constants/fishingRod/fishingRod.js`)
2. 기대단가 계산 유틸 함수 작성: `calculateExpectedFishMeatPrice(rodGrade)` → `{ 소형, 중형, 대형 }` 반환
3. 위 "계산 예시"로 유틸 함수 검증 (272.2 근사값 나오는지 확인)
4. localStorage 읽기/쓰기 헬퍼 작성 (`getFishRodSettings`, `setFishRodSettings` 등 — 기존 프로젝트에 localStorage 유틸이 이미 있다면 재사용)
5. `FishPage.jsx`에 낚싯대 선택 UI 추가 + 상태 연결
6. 계산 결과 표시 UI 추가
7. 요리/가공 페이지에서 캐싱된 기대단가를 가져다 쓰는 부분 연결

## 6. 주의사항

- 마을 구분 없이 **사이즈 기준으로만** 기대단가를 계산하고 저장한다 (4마을 모두 사이즈별 가격이 동일하므로 중복 계산 불필요).
- 등급 확률 가중치는 사용자가 선택한 낚싯대 고정 확률표를 그대로 사용하고, 별도의 정규화(합이 1인지 확인)는 상수 데이터 작성 시 한 번만 검증한다.
- 새로운 색상/스타일을 임의로 추가하지 말고 기존 디자인 시스템(버튼, 셀렉트, 카드, 표 컴포넌트)을 재사용한다.
- 커밋 메시지는 Korean conventional commit 형식 사용 (예: `feat: 낚싯대 등급 기반 생선살 기대단가 계산 기능 추가`).
